Document Object Model

The DOM is niether part of html, nor part of the javascripy; it is a seperate set of rules. It is implented by all major browser makers, and covers two primary areas.

The Dom specifies the way in which the browser should structure this model using the Dom Tree
The Dom is called an object model becasue the model (the Dom Tree) is made of objects. each object represents a different part of the page loaded in the browser window.

accessing and changing the html page

The Dom also defines methods and properties to access and update each object in this model, which in turn updates what the user sees in the browser.

Access the elements

  • select an individual element node
  • select multiple elemetns (nodelists)
  • traversing between element nodes

work with those elements

  • access/update text nodes
  • work with html content
  • access or update that attribute value
Methods thta find elements in the dom tree are called dom queries. when you need to work with an element more than once, you should use a variable to store the result in this query
when people talk about storing elements in varibales, they are really storing the location of the elements within the dom tree in a variable. the properties and methods of that elements node work on variable.
